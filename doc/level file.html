Level File Specifications

This file specifies all information needed for each level to load, from what images to use as backgrounds to where to place the enemies. Each line starts with one of several tokens which identify how to handle the information presented on that line.

Line Specification Token Element 1 Element 2 Element 3 Element 4 Element 5
Level Header Info ! Number of blocks wide. Number of blocks high. Width of each block (px). Height of each block (px).
Block specification @ Image's file name X position of block (See notes) Y position of block (See notes) Collision Code (See notes)
Player Info # Image's file name X position of player (See notes) Y position of player (See notes)
Background Layer Images $ Image's file name
Enemy Info % Image's file name X position of enemy (See notes) Y position of enemy (See notes) X velocity of enemy (See notes) Y velocity of enemy (See notes)
Comment // Comment text

Notes

"X position" and "Y position" are specified in block-widths and block heights respectivly from the upper left corner of the screen.

"X velocity" and "Y velocity" are specified in pixels per update. An update occures approximately once every 1/30 second.

The "Collision Code" is a 4 diget binary number where a one (1) represents "true" and a zero (0) represents "false". Each diget represents whether a certain side is "collidable", that is whether a player, enemy, etc. should trigger a collision event upon colliding with that side of the block. Each diget represents a side, ordered top, right, bottom, left when reading from left to right. For example a code of "1011" mean that a block is "solid" on the top, bottom, and left sides, but not the right side.

The "Level Header" line must be the first line in the file, not counting comments. All other lines may occur in any order, however good style is to group lines specifiying similar content.
